Http Not Supported or Disabled in Libcurl

Written by leiheng with ♥ on November 12, 2019 in misc , 245 words

如题. 一个这样的问题. 我在网上搜的时候发现排名第一的StackOverflow的回答说是 windows 不支持单引号 ==_==…..

想到之前自己做过的一个实验, 在win上用同一个东西加密, 单引括起来的加密的结果就是和双引号/不加引号的结果不一样, 没想到今天居然还搜到了最终回答….. win这个狗逼不支持单引号

但是我的环境是在linux 上出现的这个问题. 这个最终答案也很蠢…. 就是http 和 双引号之间有个 空格….

Like:

leih:/opt/dah # curl --noproxy localhost "http://localhost:9060/Action=Encrypted"
{"autnresponse":{"action":{"$":"QUERY"},"response":{"$":"SUCCESS"}}}
leih:/opt/dah #
leih:/opt/dah # curl --noproxy localhost " http://localhost:9060/Action=Encrypted"
curl: (1) Protocol " http" not supported or disabled in libcurl
