About proxy

  1. windows 设置 proxy 必须从环境变量 设置, 这里不是说从 powershell/cmd/gitbash 设置. 并且它可以设置protocol: HTTPS_PROXY=http://server:port 这种样式.
  2. Linux 大家都知道能够设置protocol.
  3. maven & gradle 必须从 settings.xml/gradle.properties 中读取 proxy 配置, 且不支持设置 protocol.
  4. npm & git 支持设置protocol, 且支持disable https.
  5. sbt 的 proxy首先是读取的 JAVA_OPTS, 如果里面设置了http, 那么就读这种proxy, 但是, JAVA_OPTS 不支持设置 protocol.

所以, 如果你的 proxy 不支持https 方式连接, 那么酌情混合使用.

说一说为啥我的postman总是找不到localhost和127.0.0.1吧

这也是配置的问题, 我一直觉得postman用系统proxy是走的那个网络proxy, 就是通过chrome:/settings找到的那个配置proxy的地方. 但其实不是, 他妈的去找环境变量了, 就是HTTP_PROXY这个环境变量, 所以, 你还的配置一个 NO_PROXY 这样的环境变量…..

顺便记录一下, postman dev tool 挺好用的, 打开它去看request,还能看到具体的报错信息, 解决postman proxy 的问题就是通过看他具体报错看出来的……..
