Maven Dependencies Pom Type

Written by leiheng with ♥ on August 13, 2019 in java , 351 words

终于有一天, 我看到我自己 java 练习项目依赖, 配置乱成一坨糟, 心里很糟心, 就打算整理整理整理.

于是创建了 common-dependency-parent, 项目结构如下:

.
|-- common-dep
|   `-- pom.xml
|-- pom.xml
`-- shared-pom
    `-- pom.xml

其中, shared-pom 将会作为其他 project 的parent project, 并引用 common-dep 来 import 常用的 dependencies. install之后, 尝试引用 parent 和 common-dep 失败, 并报错:

‘dependencies.dependency.version’ for com.leih:common-dep:jar

找了大半天, 为啥会去找jar, 对着 settings.xml 文件找错, offline 开了又关关了又开, 还是没有找到原因.

最后, 发现引用 common-dep 的时候忘记加 <type>pom</type> 了. 如果不显示指定依赖,那么 maven 就会自动认为这个是个 jar 包, 并从local, remote 查找.

单独拉出一个project 当作依赖项, 内容只有 pom.xml 的时候, 不指定 packaging type 是 pom 的话, 那么 mvn install 就会给你生成一个空的jar包.

所以, 如果很不幸没有指定 packaging 并且引用的时候也没有指定 type ,基本上就emmmm.

share-pom pom.xml:

    <artifactId>shared-pom</artifactId>
    <version>1.0-SNAPSHOT</version>
    <packaging>pom</packaging>
    ...
    <dependencyManagement>
      <dependencies>
      ...
        <dependency>
          <groupId>com.leih</groupId>
          <artifactId>common-dep</artifactId>
          <version>${project.version}</version>
          <type>pom</type>
        </dependency>
      </dependencies>
    </dependencyManagement>

your project module pom.xml:

    <dependency>
      <groupId>com.leih</groupId>
      <artifactId>common-dep</artifactId>
      <type>pom</type>
    </dependency>
